# Security Policy

## Reporting a Vulnerability

If you discover a security issue in NegotiationForge, please do not open a public issue with exploit details.

Instead:

- contact the maintainers privately first
- provide a clear description of the issue
- include reproduction steps, impact, and any logs that help confirm it

## What Counts as a Security Issue

Examples include:

- secret leakage
- unsafe exposure of API credentials
- authentication or authorization flaws
- prompt or data paths that expose private information unexpectedly
- dependency or deployment misconfiguration with real security impact

## Response Expectations

This is currently a small open-source project, so response times are best-effort rather than guaranteed.  
Still, valid reports will be reviewed seriously and fixes will be prioritized based on impact.
